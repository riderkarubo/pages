## 🔧 運用ルール

### 1. 新規ファイル作成時
- ファイル名末尾に日付（`_YYYYMMDD`）を付けること
- 該当カテゴリのサブフォルダに直接作成
- ルート直下（`提案骨子/`）にファイルを置かない

### 2. HTML編集時（重要）
- **`01_提案書/提案に向けた情報まとめ_統合版_20260413.html`** を編集
- git pre-commitフックが自動で `mccm-proposal/index.html` に同期
- フックが動かない場合は手動で：
  ```bash
  cp "01_年次報告会/提案骨子/01_提案書/提案に向けた情報まとめ_統合版_20260413.html" mccm-proposal/index.html
  git add mccm-proposal/index.html
  ```

### 3. ファイル命名ルール
- 分析ドキュメント: `分析_A{番号}_{テーマ}_{日付}.md`
- サマリー: `{テーマ}_サマリー_{日付}.md`
- TODO: `TODO_{用途}_{日付}.md`
- HTML: `{タイトル}_{日付}.html`

### 4. バージョン管理
- 作業中: 末尾に `_ドラフト` を付加
- 確定版は新規ファイルとして保存（古い版は `_old` を付加）
- **既存ファイルを上書き保存しない**（プロジェクトCLAUDE.mdの安全ルール）
